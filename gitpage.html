GIT

This page will give you all the details on how and what to learn about git version controler. It will also provide you a sample projects for better understanding.
Here is the pictorial view of git stages and features

There are 2 ways to maintain your code.
  1. Local repository: the files are maintained on your local system. Below are the steps to maintain the files locally using git

    • Initialise git : Create local empty git repository. ".git" file is created in the working directory. The command line Syntax is
      git init

    • Staging Area: A intermediate place to pick the files in working directory that we want to commit to git. We can add the files to stagging area by the following command line syntax:
      "git add ." - this command will pick all the files in working directory
      "git add filename.java" - this command will pick the file filename.java
      "git add *.java" - this command will pick all the files with extension .java
      "git rm --cached r ." - all the files are removed from git stagging area

    • Commit: To commit the files/changes to git local repository. The command line Syntac is
      git commit -m "add the message here in present tense"

  2. Remote repository: the files are uploaded to the GIT/github website and it can be shared with other people.
    Once the local repository is setup we can move/upload the files to github by following command line syntax
    "git remote add origin URL of github remote repository"
    "git push -u origin master" pushes the local repository to remote repository(origin) master branch using -u(links local and remote)
Here are some useful tips
  • "dir /a" - this will list all the files in the directory including the hidden files
  • "git status" - it shows the status of the untracked files in the working directory (untracked files are files that are not in stagging area)
  • "git log" - this will list all the commits till now. press q to exit the edit file
  • "git diff" - this will list the differences between the current and last commited code
  • "git checkout filename.ext" - this will rollback the file to the last commited code. Ereases all the local changes.
    "git checkout ." this will rollback all the working directory code to the last commited code.
  • on github account, navigate to insights >> graphs >> network. we see all the commit, branches and pull requests.

Gitignore

This is a hidden file in the working directory. All the files with extension listed in this file are not considered for Stagging and therefore not committed to local or remote repository. These files can be user sensitive data or user specific files.The command line syntax to create
touch .gitinore - follow the exact name as git will be searching to this file name, it is casesensitive.

Cloning

Cloning a remote repository to local working directory. The command line syntax is
"git clone URL of the remote repository"

Branching and Merging

Master - Main Branch. We can create new branch/s, write and commit code to this branch through command line or github website. Later we can merge this branch with Main branch. Resolve any conflits. The command line syntax is
"git branch branchname"
"git branch" lists all the branches and * shows which branch is the current
"git checkout branchname" changes the current branch to the branchname provided
for merge change to master branch
"git merge branchname" opens an editor to add a merge message. :q! to save and quit
"git push origin master -u" pushes to code to remote repository

Forking, Pull Request and Mergning

This is a way to collobarate and working on the project. A person who doesn't have write access to my githug project they can copy(fork) the project to thier own github account. Once they push the changes to their project, they can create a pull request to my project. I can review the changes and accept/reject the changes.
Accept - pull the changes and merge their repository to mine, new commit.
Reject - don't accept the changes
